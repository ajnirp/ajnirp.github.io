I submitted two project proposals for [Google Summer of Code](https://developers.google.com/open-source/soc/) 2015, and one was accepted!

The one that didn't get selected was an application to [improve tissue tracking utilities](https://docs.google.com/document/d/1DFujkjvKK1ZPXGsUbqMJfxEgSo9XjgIhRzZ3Oxp9DHs/edit) in [Dipy](http://nipy.org/dipy/), a research initiative to collect and implement state-of-the-art MRI algorithms. The one that did get selected was [a proposal](https://www.google-melange.com/gsoc/proposal/review/student/google/gsoc2015/rohanprinja/5741031244955648) to implement a DHCP (v4 and v6 both) client in Guile Scheme (!!!) for [GNU Guix](http://www.gnu.org/software/guix/guix.html), a functional package manager that can be used standalone in any GNU/Linux distro or as part of GuixSD, a distro under active development by the Guix team. Guix has a very interesting approach to package management that tries to solve many of the problems encountered by other package managers. I'm an Ubuntu user and I've lost count of the number of times `apt` got stuck in dependency hell, and it's *never* fun dealing with lost or broken dependencies. Here's a [blog post](http://elephly.net/posts/2015-04-17-gnu-guix.html) by a Guix contributor that talks about using Guix in an HPC environment.

I'll be using this blog to post weekly updates about my progress in implementing the DHCP client. Do follow along!
